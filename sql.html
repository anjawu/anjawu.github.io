SQL Data Analysis

As much as I loved coding in Python, learning SQL has opened up so many opportunities. After spending time coding, I have found that SQL is like Excel and Python had a baby and it came out as a super easy to use data analyzing tool. In order to learn the basic syntax of SQL, I was recommended Mode's SQL Tutorial. I LOVED it! If you are new to SQL I would definitely recommend doing it.

Below I will summarize all the syntax and skills I have learnt, along with my reports from the three sections of the hypothetical problems with fake data from the Mode Analytics Training.


Syntax

SQL Syntax

In the tabs below, you will find a summary of the terminology and tools I learnt from the Mode tutorial.

  • Basics
    Clauses

    The following is in the query clause order it must appear in (only the bold clauses must appear in each search).

    • SELECT - specify which columns to look at (* means all columns)

    • FROM - specify which table the data is being pulled from (format of: name of user.name_of_table)

    • WHERE - filter data based on a certain condition

    • ORDER BY - reorder results based on the data in columns, can be more than one column (DESC used for descending order; otherwise automatically in ascending order)

    • LIMIT - to speed up query searches, limit it to the top 100 (LIMIT 100 - Mode does it automatically)

    Note: to refer to a column use double quotations (“”) only need to do it when column name is the same as a function name in SQL (example: group)

    Clause to be used in combination with other clauses
    • AS - used for naming a new column. Example:
      SELECT west AS west_region
      FROM user.housing_info
    Logical Operators
    • AND
    • OR
    • NOT
    • IS NULL
    • BETWEEN
    • LIKE - allows search for keywords (case-sensitive)
      ILIKE - allows search for keywords (not case-sensitive)
      % is a wildcard character that allows searching for strings with any character or set of characters contained where the % is located

    • IN - look at a list of values rather than writing them as separate statements. It can be for any data type. Example:
      WHERE
      category IN (‘item one’, ‘item two’, ‘item three’)

    • BETWEEN - look at a range of values, it can be numerical or a string. Example:
      WHERE
      name BETWEEN ‘a’
      AND ‘m’
  • Intermediate
    Clauses

    Continuing the query clause order (from the basics section)

    • SELECT
    • DISTINCT - allows the user to see the unique values in a particular column (for two or more columns then they are listed as ordered pairs). It can be used in combination with aggregate functions - primarily:
      COUNT(DISTINCT month)
    • CASE - is like an if/then statement:
      WHEN - if this condition is true
      THEN - then we will make it this value (can be string, number or boolean)
      ELSE - not necessary but use if needed to encompass the rest of the data
      END - you must always end the case statement
      Two ways to approach it using aggregate functions:
      1. with aggregate function:
        SELECT
        CASE WHEN location = 'west' THEN 'west'
        WHEN location = 'east' THEN 'east'
        END AS location_group,
        COUNT(1) AS count
        FROM user.housing_info GROUP BY 1
      2. inside aggregate function (this pivots the table above) - must put the case statement within the aggregate function. Example:
        SELECT
        COUNT(CASE WHEN location = 'west' THEN 1 END) AS west_count,
        COUNT(CASE WHEN location = 'east' THEN 1 END) AS east_count,
        FROM user.housing_info
    • FROM
    • JOIN - let’s the user combine different (or the same) tables
      INNER JOIN (can be written as just JOIN) - is the intersection between the two tables; any rows that are not in both tables will be disregarded
      Outer joins - allows inclusion of rows that are not found in both tables. Broken up into:
      LEFT JOIN (or OUTER LEFT JOIN) - returns unmatched rows from left table (first one listed)
      RIGHT JOIN (or OUTER RIGHT JOIN)- returns unmatched rows from the right table (second one listed). This is not used often because you can get the same results using LEFT JOIN if you switch the order of the tables listed in FROM
      FULL JOIN (or FULL OUTER JOIN) - returns unmatched rows from both tables; gives the union of the two tables with matching rows being overlapped. This is not used very often but Mode has suggested the most common use is if it is used in conjunction with aggregations to understand the amount of overlap between two tables
      CROSS JOIN - allows the joining of two tables which are not related to each other. This will produce a table that has every permutation of rows from each table
      Note: You can give a table an alias by typing the variable after a space beside the table name
      Joining with multiple keys - this can add accuracy (and sometimes speed) to the query. AND is used with sets of keys being defined in order to join with multiple keys

      ON - says which columns should be used to map the tables (the columns are called: "foreign keys" or "join keys"). This must be included with JOIN for any inner joins.
      Note: any conditions put in the ON clause will be applied to the separate tables before the joining of them

    • UNION - allows the user to stack two tables one on top of the other. Different from FULL JOIN because it does not look for similarities based on common values within a column, but will drop full rows in the second table if they are identical to the first table. In order to use UNION, the following two conditions must be satisfied: both tables must have the same number of columns, and the columns must have the same data types in the same order as the first table
      UNION ALL - is used to append tables together without any rows being dropped (even if the rows are the same)

    • WHERE (using JOIN) - will apply the condition(s) after the join; don’t forget WHERE can filter NULL values as a byproduct unintentionally

    • GROUP BY - allows user to group data based on a specific column(s) (must be used in conjunction with an aggregate function)

    • HAVING - like WHERE except allows you to filter on aggregate columns

    • ORDER BY
    • LIMIT
    Aggregate functions
    • COUNT counts how many rows are in a particular column
    • SUM adds together all the values in a particular column
    • MIN and MAX return the lowest and highest values in a particular column
    • AVG calculates the average of a group of selected values
