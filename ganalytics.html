BigQuery

BigQuery

BigQuery is a data warehouse. Since it is Google's they have a lot of other products that can be connected, it lets you scale your data and really build a good project for free. Google offers various tiers for using their data warehouse. As someone working on personal projects, I am using the free tier. I found this to be the best resource to use to practice SQL queries on my own data. There are SQL servers that will let you input your own data, but none that I had explored was as easy to set up and free like BigQuery.

  • SQL Queries
    Create Table & SAFE_

    • W3Schools has a good page that talks about ways to create tables. Since I already went through how to make a new dataset (with a schema) in “Setting up Dataset” drop down section, here I wanted to go through creating a new table (from an existing one) and changing some data types.

      To create a new table from an existing one, use “AS” and “SELECT” what you want “FROM” an existing table.

      In BigQuery you can use the SAFE prefix. According to the documentation: if you begin a function with the SAFE. prefix, it will return NULL instead of an error.

      Here is the sample code from my school board achievement and progress table:

      CREATE TABLE
      school_board.achievement_progress AS
      SELECT
      p.Board_Number AS board_number,
      p.Board_Name AS board_name,
      SAFE_CAST(p.Grade_6_EQAO_Reading_Results AS FLOAT64)*100 AS grade_6_EQAO_reading_results_percent,
      SAFE_CAST(p.Progress_in_Grade_6_EQAO_Reading_Results AS FLOAT64)*100 AS progress_in_grade_6_EQAO_reading_results_percent,
      p.Extract_Date AS extract_date
      FROM
      school_board.progress p

      As you may have noticed, I chose to shorten the name of my previous table to “p” to make it simpler to reference the columns I wanted.

  • Data Studio Academy Course

    I am the type of person who likes to explore and try things out, so I wasn’t a huge fan of this academy course. It gave a lot of information, most of which I did not find that useful. Below you will find the bits that I pulled out from the whole course that I found useful for myself.

    Definitions
    • Dimensions: data categories that can be any type value (including unaggregated numbers)
    • Metrics: calculations of various aspects of dimensions (aggregated numbers)
    • Data set: data living outside of Data Studio
    • Data source: Data Studio’s representation of the data set(s)
    • A bullet chart (or “thermometer graph”): visualization for comparing a given metrics performance against target benchmarks (usually used for KPIs)

    I had no problem understanding how sharing view/edit rights for the report work - as it is like all other Google products - but for some reason I had a hard time wrapping my brain around credentials.

    Credentials
    These are just for the data source, not the report.
    • Owner’s credentials: this needs to be set if the people who will be viewing the report DO NOT have their own access to the data set.
    • Viewer’s credentials: this needs to be set if the people who will be viewing the report DO have their own access to the data set.

    The most useful information I got out of the whole course was section 4.1:

    Foundational principles of visual design
    1. Start with a quick simple overview of the big picture of the report.
    2. Then, allow them to help themselves to more detail as needed. This might come in the form of filter controls, additional report pages, or hyperlinks that will load more detail when the viewer is ready for it.
    3. Visual elements that are similar in shape, size, and/or color are perceived to be more closely related.
    4. Color is a powerful tool to guide viewers through your report.
      • Use transparency to create visual hierarchy: A sliding scale allows you to make component colors more or less transparent.
      • Use saturation to draw attention: Intense colors draw the eye, so use to highlight key aspects.
      • Don’t rely on color: Too much colour can detract from the main focus points. Some people might also have trouble reading visualizations with too much colour.
    5. Pick (or create) Theme inorder to ensure that the report is stylized the same throughout. It also includes a setting for how colors should apply consistently:
      • By series order: The color depends on the order of the item in a series.
      • By dimension values: The same value maintains a fixed color.
      • Finally, you can also choose colors to represent increasing or decreasing metrics across your report.

    Template Use
    If you plan on using your report layout more than once, you can:
    • Simply copy a report OR
    • Create a template from the report: Simply take the URL of the report, and append “/preview” to the end of the URL. This will prompt the report to appear with the “USE AS TEMPLATE” button, and click yes

  • Adding Time on Page Tracker

    As I used GA for my website, I soon got very frustrated about the view times I was seeing (majority of the time being an average of 0 seconds). Since this seemed like way too small of a number I decided to check out if other people had this experience. Sure enough, a quick search led me to finding that if someone only visited one page and then left, GA would not be able to tell me the time they spent on the page. This was due to the set up of the monitoring by GA, where an action (such as clicking on a new page) must happen for GA to pick an end time from the entrance point.

    Since I really wanted to know how long people spent on each page of my website, I spent quite a bit of time researching what to do. After my research, I decided the solution I liked the most was from a blog post I found.

    Side note: I did try the other code that they mentioned in the blog, but the website link no longer existed and the copying of the code did not work for me.

    All I did was add the following code to my GA code at the top of my pages (after my GA global site tag < /script>):

    < script type="text/javascript">
    function timer11(){gtag('event','1', {'event_category':'pagename_TimeOnPage','event_label':'11-30 seconds','non_interaction':true});}
    function timer31(){gtag('event','2', {'event_category':'pagename_TimeOnPage','event_label':'31-60 seconds','non_interaction':true});}
    function timer61(){gtag('event','3', {'event_category':'pagename_TimeOnPage','event_label':'61-180 seconds','non_interaction':true});}
    function timer181(){gtag('event','4', {'event_category':'pagename_TimeOnPage','event_label':'181-360 seconds','non_interaction':true});}
    function timer361(){gtag('event','5', {'event_category':'pagename_TimeOnPage','event_label':'361-600 seconds','non_interaction':true});}
    function timer601(){gtag('event','6', {'event_category':'pagename_TimeOnPage','event_label':'601-1800 seconds','non_interaction':true});}
    function timer1801(){gtag('event','7', {'event_category':'pagename_TimeOnPage','event_label':'1801+ seconds','non_interaction':true});}
    gtag('event','0', {'event_category':'pagename_TimeOnPage','event_label':'0-10 seconds','non_interaction':true});
    setTimeout(timer11,11000);
    setTimeout(timer31,31000);
    setTimeout(timer61,61000);
    setTimeout(timer181,181000);
    setTimeout(timer361,361000);
    setTimeout(timer601,601000);
    setTimeout(timer1801,1801000);
    < /script>

    Go to “Admin”

    Click “All Filters”

    → +Add Filter

    In order to remove your specific IP address, select:

    • Exclude
    • traffic from the IP addresses
    • that are equal to
    • Select the webpages you want to exclude yourself from and move them to “Selected views”

    Now enter your IP address. I found the best way to exclude your devices was by looking up what your public IP is. You can do this by simply googling “what is my IP address”.
